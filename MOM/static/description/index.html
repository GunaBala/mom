Internal Helpdesk



Overview

This module helps to provide and share the required information between the departments/users within the organization. Users can create any type of request and send it to concern department/users to process the request.

Features

Dashboard - My requests and Others requests

A request can be sent to all users or specific users within the department.

Priority-based SLA

Add or remove the access to the users for the particular request.

Document attachments

Create and store the details relevant to the request like images, table content or other text descriptions etc.

Traceability details

Discussions within the request

Send Request

Once the request has been sent, the status moved to request sent. In ticket allowed users field, we can see the list of users who have access for this request.


Users Dashboard

In Dashboard, User can see overall request status-wise, the request from others will be shown under Other Requests and users own request details will be shown under Requested By Me.

Under overdue, it will show the request which is not responded on time based on the SLA mapped with the priority.


Process request to respond - Update required information like upload attachments and resolution update

Any allowed users can send the response for the request, user can attach the files and update the required details.


Providing access to multiple users to process the request

Any allowed user can add users to the request, once the user is added, that user allowed to access the request.


Users added and request responded

The requested user can Reopen/Resolve the request

The reopen and resolve button will be visible for the request created user. The user can resolve or reopen the request.
